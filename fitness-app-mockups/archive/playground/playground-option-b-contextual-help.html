When Can Tokens Be Accessed?
This controls when your app can read the tokens from the keychain:

‚Ä¢ After First Unlock: Once user unlocks device after restart. Best for most apps - background refresh works.
‚Ä¢ When Unlocked: Only while device is actively unlocked. More secure but breaks background tasks.
‚Ä¢ This Device Only: Tokens won't be in iCloud backups - extra security layer.
For a fitness app, use "After First Unlock" so background sync works properly.

Additional Security Layers
Biometrics: Requires Face ID/Touch ID every time the token is accessed. Good for banking apps, overkill for fitness apps.

iCloud Sync: Tokens available on all user's devices. End-to-end encrypted, but increases attack surface.

Access Group: Share tokens with your app extensions (Watch, Widget, Share). Only enable if needed.

Migrating Existing Users
If your app currently stores tokens in UserDefaults, you need to migrate existing users to avoid logging them out.

The migration will:
1. Check for tokens in UserDefaults
2. Copy them to Keychain (encrypted)
3. Delete the plaintext copies
Always enable this unless you're building a brand new app with no existing users.
